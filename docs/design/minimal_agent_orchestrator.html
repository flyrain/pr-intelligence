Minimal Agent Orchestrator Kernel (POC Design)

Overview

This document describes a minimal agent orchestration kernel that separates work definition, control logic, and execution runtime. Agents are treated as stateless executors, while all durable state is maintained externally.

Design Goals

  • Durable task state independent of agent sessions
  • Stateless, replaceable execution
  • Simple, inspectable control loop
  • Minimal abstractions

Core Principles

  • Externalized state
  • Tasks over sessions
  • Event-driven system
  • Control plane separation

Core Abstractions

{

  "status": "ready",

  "id": "exec_1",

Control Loop

def tick():
    tasks = task_store.find(status="ready")
    for task in tasks:
        assign_and_start(task)

    executions = execution_store.active()
    for exec in executions:
        result = runtime.poll(exec.id)
        handle_result(exec, result)

State Model

created → ready → running → completed
                    ↓
                blocked → ready
                    ↓
                  failed

Persistence Model

Tasks Table

id, title, status, input, owner, attempt_count

Executions Table

id, task_id, agent_id, runtime, status

Events Table

id, task_id, type, timestamp, payload

Key Invariants

  • Task state is authoritative
  • Events are append-only
  • Execution is replaceable
  • Orchestrator is stateless

Summary

The kernel consists of Task, TaskGroup, EventLog, RuntimeAdapter, and Orchestrator. It enforces external state, stateless execution, and a policy-driven control loop.
